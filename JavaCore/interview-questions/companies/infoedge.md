# InfoEdge — focused on Elasticsearch primarily

**Total Rounds:** 2 (1 hour each)

## Questions
1. Tell me about your current project.

2. Have you used Elasticsearch in your project? If yes, describe your usage briefly.

3. What is Elasticsearch?

4. What makes Elasticsearch fast?

5. How do shards and replicas work? What are typical shard/replica/RAM sizing considerations?

6. The search API that internally calls `_search` times out because 
   - Elasticsearch is slow — how would you troubleshoot?

7. How does master node and data node work here, 
   - based on search how these nodes operate. 
   - Further based on number of shards and replicas how these elastic search nodes behave?

8. Any particular reason for frequent company changes?
